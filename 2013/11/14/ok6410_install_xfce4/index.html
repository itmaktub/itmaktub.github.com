xfce4编译与安装

By itmaktub

2013-11-14 更新日期:2014-09-07

文章目錄

　　想给我的的OK6410装个x，思来思去，最终挑选了轻量级的xfce4，

　　因为已经给Ok6410移植了debian，是可以apt-get install xfce4的，但本着不折腾就会死的心理，决定现在PC上手动编译一下，记录编译过程

  1. 下载并解压源码

我直接省事，把所有的组件都打包下载了，也可以自己选择需要的组件编译

http://archive.xfce.org/xfce/4.10/fat_tarballs/xfce-4.10.tar.bz2

tar xvjf Download/xfce-4.10.tar.bz2 -C application/

cd application/src

rm *.md5 *.sha1

tar xvjf xxx.tar.bz2依次解压各压缩包

2.按顺序编译

因为各种包和库的依赖，所以必须按照一定的顺序来进行

xfce4-dev-tools, libxfce4util, xfconf, libxfce4ui, garcon, exo, xfce4-panel, Thunar, thunar-volman, xfce4-settings,
xfce4-session, xfdesktop, xfwm4, xfce4-appfinder,xfce4-power-manager,gtk-xfce-engine,tumbler

分别进入各个文件夹执行sudo ./configure&amp;&amp;sudo make &amp;&amp;sudo make install(配置的时候不使用root会报/usr/share格式错误)

中间会提示缺少库之类的,例如:

configure: error: Package requirements (glib-2.0 &gt;= 2.24.0) were not met:

No package 'glib-2.0' found

或者

checking for intltool &gt;= 0.31... found

configure: error: Your intltool is too old. You need intltool 0.31 or later

此时sudo apt-get install xxxxx即可

我装的大概有这些：（不完全统计，因机而异）

sudo apt-get install libglib2.0-dev intlttool libdbus-1-dev libgtk+2.0-dev libwnck-dev libgudev-1.0-dev x-org

　　编译速度也很快，执行完之后退出原来的gnome桌面，在会话登录的时候就可以看到xfce4了，如果提示启动出错，找到那个库文件相关的组件，从新执行编译安装会解决一部分问题，如果提示缺少文件，请自行安装，

　　话说执行完后，感觉速度很快，就决定把gnome删掉，先是sudo apt-get remove gnome，只删除了一个100多k的文件，然后自动补全提示还有好几百个，就手贱用了autoremove，好多软件都木有了啊！当时想死的心都有，上网下载不容易啊！！！xfce不准备收拾了，尝试用几天文本模式吧，努力提升逼格ing。。。。
